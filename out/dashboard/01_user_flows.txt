USER FLOWS - DASHBOARD MOCK PHM

Flow A: Manual prediction
1) User selects dataset_id, unit_id, cycle.
2) User adjusts op_settings and optional sensor values.
3) User clicks "Run Prediction".
4) App shows loading state.
5) App renders RUL estimate, risk level, confidence band and recommendation.

Flow B: CSV prediction
1) User switches input mode to CSV.
2) User uploads CSV with required columns.
3) App validates schema and reports missing columns if any.
4) User chooses row index.
5) User clicks "Run Prediction".
6) App renders outputs and historical trend.

Flow C: Error handling
1) User sends invalid dataset/unit/cycle/range values.
2) Service returns validation error.
3) App shows error_validacion state with explicit message.

Flow D: No data
1) User opens app without inputs.
2) App shows sin_datos state and expected next action.
