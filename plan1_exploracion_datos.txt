PLAN DE TRABAJO - EXPLORACION DE DATOS CMAPSS

Objetivo general
- Entender calidad, estructura y comportamiento temporal de CMAPSS para definir un pipeline robusto de modelado RUL.

Alcance
- Subsets: FD001, FD002, FD003, FD004
- Archivos de entrada: data/train_*.txt, data/test_*.txt, data/RUL_*.txt
- Archivos para visualizacion ya disponibles: out/excel/train.xlsx, out/excel/test.xlsx, out/excel/rul.xlsx

Precondiciones
1) Activar entorno conda:
   conda activate cmapss
2) Verificar librerias:
   python -c "import pandas,numpy,matplotlib,scipy,openpyxl; print('ok')"
3) Crear carpetas de salida:
   out/eda
   fig/eda

FASE 1 - Inventario y validacion estructural
Objetivo
- Confirmar integridad basica y consistencia de formato.

Tareas
1) Contar filas por archivo y por subset.
2) Confirmar numero de columnas esperado (26 en train/test).
3) Verificar que no existan valores nulos inesperados.
4) Revisar tipos de datos y rangos anormales.
5) Validar correspondencia entre:
   - numero de motores en test
   - numero de etiquetas en RUL

Entregables
- out/eda/01_inventory_summary.csv
- out/eda/01_schema_report.txt

Criterio de salida
- Estructura valida en los 4 subsets sin inconsistencias bloqueantes.

FASE 2 - Perfil estadistico por variable
Objetivo
- Caracterizar distribuciones, dispersion y outliers.

Tareas
1) Calcular estadisticos por columna (media, std, min, max, percentiles).
2) Separar analisis por:
   - condiciones operativas (op_setting_1..3)
   - sensores (sensor_1..21)
3) Detectar sensores constantes o casi constantes.
4) Medir correlacion entre sensores (matriz Pearson/Spearman).

Entregables
- out/eda/02_stats_global.csv
- out/eda/02_stats_by_dataset.csv
- fig/eda/correlation_heatmap_fd001.png ... fd004.png

Criterio de salida
- Lista clara de sensores utiles, redundantes y sospechosos.

FASE 3 - Analisis temporal por unidad
Objetivo
- Entender patron de degradacion a lo largo del ciclo.

Tareas
1) Graficar trayectorias de sensores por unit_id (muestra representativa).
2) Medir longitud de secuencias por motor.
3) Analizar tendencia temporal de sensores clave.
4) Verificar si hay cambios de regimen por condiciones operativas.

Entregables
- out/eda/03_sequence_lengths.csv
- fig/eda/sensor_trends_fd001.png ... fd004.png
- fig/eda/unit_examples_fd001.png ... fd004.png

Criterio de salida
- Definidos sensores con patron de degradacion y ventanas de observacion candidatas.

FASE 4 - Analisis especifico para RUL
Objetivo
- Preparar criterios para target engineering y evaluacion.

Tareas
1) Construir RUL en train por unidad (RUL = max_cycle - cycle).
2) Evaluar distribucion de RUL (train y true RUL de test).
3) Probar estrategia de capping (ejemplo: 125/130 ciclos) y comparar distribuciones.
4) Definir variable objetivo final:
   - RUL lineal
   - RUL capped
   - clasificacion de riesgo (opcional)

Entregables
- out/eda/04_rul_distribution.csv
- fig/eda/rul_histograms.png
- out/eda/04_target_definition.txt

Criterio de salida
- Definicion de target aprobada para entrenamiento.

FASE 5 - Preprocesamiento base para modelos
Objetivo
- Dejar un baseline reproducible de features para ML.

Tareas
1) Definir columnas finales de entrada.
2) Normalizar por dataset o global (justificar decision).
3) Crear ventanas temporales para modelos secuenciales.
4) Generar splits de validacion sin fuga temporal:
   - por unit_id
   - respetando orden temporal
5) Exportar datasets procesados para entrenamiento.

Entregables
- out/eda/05_feature_list.txt
- out/eda/05_preprocessing_config.json
- out/processed/train_processed.parquet
- out/processed/valid_processed.parquet
- out/processed/test_processed.parquet

Criterio de salida
- Dataset listo para baseline ML y reproducible.

FASE 6 - Control de calidad y cierre de EDA
Objetivo
- Cerrar exploracion con decisiones trazables.

Tareas
1) Resumir hallazgos tecnicos principales.
2) Listar riesgos de datos (ruido, drift entre subsets, sensores poco informativos).
3) Proponer baseline de modelado inicial (sin entrenar aun).
4) Definir backlog de mejoras.

Entregables
- out/eda/06_findings_summary.md
- out/eda/06_risks_and_actions.md
- out/eda/06_baseline_plan.md

Criterio de salida
- Documento de handoff listo para iniciar entrenamiento.

4) Fase 4

6) Fase 6

Tiempo estimado (equipo pequeno, 1-2 personas)
